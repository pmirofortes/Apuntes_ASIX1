Direcciones IPv4

Estructura de la dirección IPv4

La IP sirve para identificar un dispositivo/host en la red. Tambien se le llama dirección lógica y es de 32 bits. Es una dirección jerárquica, hay una parte de la dirección que da información, se divide en: red/host.

Direccionamiento dinámico y estático

Dinamico: La asignación de los parámetros de red (IP/Mascara/Gateway/DNS) se realiza de manera automática a través de un servidor DHCP (Dynamic Host Configuration Protocol).

Ventajas: No requiere intervención, mayor velocidad en la configuración de los equipos, se eliminan las IPs duplicadas o erróneas.

Estático: La asignación de los parámetros de red se realiza de manera manual por el administrador de red.

Direcciones IPs especiales no asignables

ID red/subred

  • IPs que tienen todos los bits de host a 0.
  • IPs de broadcast, se utilizan como IP destino a todos los paquetes de un host de una red. Todos los bits de host a 1
  • IP de localhost (127.x.x.x)
  • IP de ruta predeterminada, identifican cualquier paquete en una ruta por defecto (0.X.X.X). La 0.0.0.0 es la que tienen los host cuando está a la espera de recibir la IP del switch.
  • IP de enlace local, son las que asigna de manera automática el sistema operativo cuando no responde el servidor DHCP 169.254.X.X
  • Clase D, son direcciones multicast para un conjunto de equipos.
  • Clase E, son direcciones experimentales para grupos de desarrollo de internet (IETF)
  • Test-net, son direcciones reservadas para probar o enseñar.
