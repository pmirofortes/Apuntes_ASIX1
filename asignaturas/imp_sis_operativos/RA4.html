1. Concenptos básicos alrededor de los dominios

Dominio - Es la estructura básica organizativa de seguridad de un servicio de directorio

Servicio de directorio - Es el encargado de almacenar información sobre todos los recursos de red como usuarios, grupos, equipos, carpetas o aplicaciones.

Controlador de dominio - Es el equipo que dispone de un sistema operativo servidor promocionado (Primary Domain Controller).

Catalogo global - Es la base de datos donde se almacena toda la información de los objetos del dominio.

Esquema - Es como se representa de manera visual de los recursos de un dominio.

Objeto - Es el elemento contenido de un dominio en el servicio directorio que representa un recurso.

Atributos - Sirve para identificar una clase de objeto (nombre, apellidos, departamento)

Atributo clave - Es el atributo que sirve para identificar de forma inequívoca a una clase de objeto

Unidad organizativa - Es un objeto donde se categorizan diferentes objetos en función del departamento o de la clase (por ejemplo, todos los objetos relacionados con contabilidad o todos los usuarios)

Árbol - Es una organización jerárquica de varios dominios donde un padre tiene subdominios, formando un árbol

Bosque - Es la estructura donde hay 2+ árboles conectados. El subdominio debe llevar el apellido del padre.

Relación de confianza transitiva - Permite que desde cualquier dominio o subdominio puedes acceder a la información de otro dominio o subdominio

LDAP - Es el protocolo que utilizaremos en los servicios de directorio (LightWeight Directory Access Protocol)

2. Organización de un dominio

Dentro de la estructura de un dominio existe un catalogo global (base de datos) donde se almacena toda la información de los diferentes objetos

Cada clase de objeto tiene asignado un atributo, a través de ese atributo es sistema es capaz de daber que objeto es. Por ejemplo un objeto de clase usuario tendrá atributos como nombre, apellidos, departamento... y uno de clase equipo tendrá sistema operativo, IP, DNS...

Estrucutra lógica

Cuando podemos distinguir los objetos dentro del dominio a través de sus atributos y atributos clave, debemos estructurar dentro del dominio para que sea mas fácil organizarlos >> La unidad básica de organización es la Unidad Organizativa (OU), son contenedores de objetos >>> Las OU son objetos que nos permiten una delegación de funciones de administración por parte del administrador del dominio >>> La delegación de funciones quiere decir que el administrador del dominio puede delegar sobre otros usuarios la responsabilidad de gestionar una OU concreta, o alguna clase de objetos de una OU concreta.

3. Organización con múltiples dominios

Útiles en redes donde la administración se realiza por diferentes autoridades. > Para trabajar con dominios múltiples extenderemos el dominio padre hacia otros dominios hijos o subdominios dentro de una estructura de árbol como una estructura jerárquica de nombre DNS.
