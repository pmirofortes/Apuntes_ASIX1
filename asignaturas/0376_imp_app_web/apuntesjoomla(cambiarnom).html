3. Tipos de Usuarios

Front-End

  • Registrados: Estos usuarios tienen la capacidad de ver los artículos, pero no pueden interactuar con ellos de ninguna otra forma.
  • Author: Los usuarios con este rol pueden crear nuevos artículos y modificar los artículos que han publicado y son de su propiedad, pero no los de otros.
  • Editor: Los usuarios con este rol tienen la capacidad de modificar cualquier artículo.
  • Publisher: Los usuarios con este rol pueden cambiar parámetros clave de cualquier artículo, como el estado de publicación (por ejemplo, cambiar de "publicado" a "no publicado") o modificar su categoría.

Back-End

  • Superadmin (el Dios de la web): Es el usuario con acceso completo y control total sobre el sistema. Este rol tiene autoridad absoluta y puede gestionar todos los aspectos del sistema.
  • Administrador: Los administradores gestionan los usuarios, pero no pueden modificar ni al Superadmin ni a otros administradores.
  • Gestor: Los gestores tienen acceso para gestionar el contenido y las tareas dentro del sistema, pero no poseen control total sobre usuarios o configuraciones de alto nivel.

Ejemplo de Creación de Usuarios, Grupo, Nivel de Acceso y Menú

En este ejemplo, creamos un usuario, un grupo, un nivel de acceso y un menú oculto asignado específicamente a los usuarios de ese grupo:

  • Usuario: Se crea un usuario llamado "Autor 1".
  • Grupo: El usuario "Autor 1" pertenece al grupo "AutorASIX1".
  • Nivel de Acceso: El nivel de acceso asignado a este grupo es el correspondiente al grupo "AutorASIX1", lo que determina que el usuario tenga permisos limitados según las funciones definidas para ese grupo.
  • Menú: Se crea un nuevo menú oculto (por ejemplo, una página o sección del sistema) al cual solo pueden acceder los usuarios del grupo "AutorASIX1". Este menú estará oculto para otros grupos o usuarios sin el nivel de acceso adecuado.
